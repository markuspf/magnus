Naming conventions

Public identifiers should be as self-documenting as possible. This is a matter of taste and judgement: we considered FPGroup to be too terse, writing FinitelyPresentedGroup instead, but this made source lines too long to read.

Our spelling conventions for public identifiers are:

  • Type and class names start with upper case.
  • Names of constants are in all capitals.
  • Member and method names start with lower case, but significant subwords start with upper case. No underscores.

Local identifiers need not be as descriptive as public ones, but should still convey their meaning and use.
