Hot text

The FE accepts the message



#1 key datum

which tells it to associate datum with key in the same sense as the properties argument of the create_ message. The syntax and semantics are the same.

Many of the messages in the following sections may contain substrings which we call Hot text. This means that the substring may contain substrings of the form [_value_ key], which evaluate to the datum most recently associated with the key in a properties argument of a create_ message, or a define_ message, or the value 0 if there is no such association. Hot text may also contain substrings [_name_ expr], where expr evaluates to an objectID. This evaluates to the end user supplied name of the object.

Hot text may not contain characters [] except as above, "" except as outer delimiters, or ${} except as part of variable references or delimiters, discussed below.
