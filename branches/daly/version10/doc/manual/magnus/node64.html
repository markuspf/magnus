The Problem View

The FE provides a procedure named problemView for use as a viewBuilder in a viewStructure argument to the create_ command (§2.3.1.1). The viewParameters it expects has the form:

{viewID viewTitle viewBanner totalARCs subproblems}

The viewID is a positive integer which is the definitive reference shared by the FE and SM to the problem view, but parts of the view are independently addressable via other id numbers.

The viewTitle is Hot text, delimited by {}, which titles the problem.

The viewBanner is Hot text, delimited by {}, which describes the problem. The view sizes its banner to display all of viewBanner by counting newlines.

The totalARCs has the form {ARCSlotID value}, where ARCSlotID is a positive integer which uniquely references a slot for the total number of ARCs consumed by the problem, and value is its initial value. At present, the ARCSlotID may as well be the objectID of the problem; as there are no facilities in the SM (that I can see) for sending ARC updates to this ARCSlotID, the FE computes the new value when it gets other ARC updates.

The subproblems has the form {{probViewID name state ARCSlotID value}+}, where probViewID is a positive integer which uniquely references the subproblem, name is Hot text which describes the subproblem, state is 1 or 0 to indicate enabled or disabled, ARCSlotID is a positive integer which uniquely references the slot for the ARCs allocated to the subproblem, and value is the initial ARC value.
