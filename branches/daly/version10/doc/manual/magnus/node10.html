Special Restrictions

This section explains miscellaneous requirements and restrictions of the system.

  • The final version of your code must never write to cout. The session manager uses cout for communication with the front end, and would be confused by any unofficial chatter. This restriction does not apply to test programs.

  • Your debugging code, compiled conditionally on DEBUG, may write to cerr, but all error messages and warnings intended for the production version should be passed to the global functions error(char*) and warn(char*), resp.
