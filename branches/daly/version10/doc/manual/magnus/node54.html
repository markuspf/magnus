Messages

The FE and SM exchange messages. The syntax and semantics of the messages which the FE can accept are fixed; they are described in this section.

In contrast, the FE knows as little as possible about the messages it sends to the SM. Rather, the FE is supplied by the SM with message templates for each kind of message it will need to send. These templates contain formal parameters which the FE replaces, at the time the message is sent, with actual parameters. The syntax and semantics of formal parameters are also fixed, and are described below along with each message type.

A message from the SM to the FE is an ASCII string containing no non-printable characters, newlines, linefeeds, or tabs. It consists of a message name followed by an ordered sequence of zero or more arguments. A message name matches '[a-zA-Z][a-zA-Z_]*_'. Each argument is preceded by at least one space. The argument syntax is described with each message, below.

The following sections describe each message that the FE can accept, grouped by function. Because of the semantic interconnections between the various messages, the sections are not in the temporal order in which the messages are sent.

Except where noted, the arguments of messages to the FE are keyword arguments. For example, suppose the message named foo_ takes two arguments: an integer and a constant string. The syntax is specified like:



#1 theInt theString

In this case theInt and theString are keyword identifiers, so the actual message could be, for example:

foo_ theInt 7 theString {Hello world}

foo_ theString {Hello world} theInt {7}
