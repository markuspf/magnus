proc framer

The proc framer in general.tcl relieves some of the tedium of creating frames and packing widgets into them, though you still have to think about how to arrange frames to get the effect you want. The above example becomes 

  button .d9.ok -text OK; button .d9.cancel -text Cancel
  framer .d9 .d9 top {
    left {} {} {
      ok {} cancel {}
    }
  }

Look in, e.g., options.tcl for more elaborate examples.

The first argument to framer, parent, is the parent of all widgets which are to be packed. It saves having to re-type it for every widget. The second argument is the highest level master of all widgets packed by framer, and of all frames it creates and packs. It may as well be parent. The third argument gives the initial -side argument to pack. What follows is a list of 2- and 4-tuples.

The 2-tuples are widget names followed by a list of arguments to pack, possibly {}. These widgets must have parent parent.

The 4-tuples specify frames which may contain slave frames and widgets. They consist of a -side option for the slaves, a list of frame options, a list of pack options for the frame, and a further list of widgets and frame specifications.

As framer creates the specified frames, it gives them the names f1, f2, etc., and makes parent their parent.
