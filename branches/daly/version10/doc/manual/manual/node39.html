Global Message Templates

The FE responds to a number of events by sending essentially the same message to the SM. The SM may initialize templates for these messsages at startup with the following messages to the FE. The message is always delimited by {} if it contains whitespace:



#1 message

The message is to be sent to the SM when the end user wishes to quit. The front end does not exit until it gets the confirm_quit_ message, described below.



#1 message

The message is to be sent to the SM when the end user requests a view on a subproblem (see §3.3.3). It is Hot text, and may contain the references ${viewID} to the enclosing view, and ${probViewID} to the subproblem for which the view was requested.



#1 message

The message is to be sent to the SM when the ARCs given to any subproblem may have been changed by the end user. It can only be sent when the enclosing problem is in a suspended state, or has not been started yet. It is Hot text, and may contain the references ${probViewID}, ${ARCSlotID}, ${value}.



#1 message

The message is to be sent to the SM when the end user starts a problem. It is Hot text, and may contain the reference ${viewID} to the viewID of the problem.



#1 message

The message is to be sent to the SM when the end user suspends a problem. It is Hot text, and may contain the reference ${viewID} to the viewID of the problem.



#1 message

The message is to be sent to the SM when the end user resumes a problem. It is Hot text, and may contain the reference ${viewID} to the viewID of the problem.



#1 message

The message is to be sent to the SM when the end user terminates a problem. It is Hot text, and may contain the reference ${viewID} to the viewID of the problem.



#1 message

The message is the prefix of the message to be sent to the SM when the end user deletes one or more objects. The FE appends the objectIDs of the objects to be deleted, separated by spaces. The object list is closed under dependency.
